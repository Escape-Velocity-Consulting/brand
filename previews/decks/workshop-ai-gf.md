## Demo 1 — Vom Sprachmemo zur Marktrecherche

### Use Case
"Neue Idee — aber wie steht die Konkurrenz?"

### Features
**Voice Input** · **Context (Files & Bilder)** · **Deep Research**

### Live
Strategie-Gedanke einsprechen → AI als Sparringpartner → PDF und Screenshot droppen → Wissensgrenze stößt → Deep Research startet, läuft im Hintergrund.

===

<!-- @type: content -->
## Demo 2 — Calls vor- und nachbereiten

### Use Case
"Kundencall in einer Stunde, kein Kopf zum Mitschreiben."

### Features
**Voice Input** · **Connector Wissensdatenbank** · **Projects**

### Live
Connector zieht Kunden-Kontext aus der Wissensdatenbank → Briefing entsteht → nach dem Call Voice-Note als Debrief → AI strukturiert → Connector schreibt zurück.

===

<!-- @type: content -->
## Demo 3 — Der automatische Morgen-Brief

### Use Case
"Den Tag starten ohne 20 Minuten Tool-Hopping."

### Features
**Connectors** · **Skills**

### Live
Connectors mit Kalender und Tasks → Fibery lesen → `/morning-brief` Skill aufrufen → konsolidierter Tagesdigest.

===

<!-- @type: content -->
## Demo 4 — Prozess rekonstruieren und konservieren

### Use Case
"Jahresprozess, vergessener Ablauf, nächste Woche fällig."

### Features
**Cowork** · **File Drop / Connector** · **Skill Creator**

### Live
Cowork bekommt letztjährigen Buchhaltungs-Folder → rekonstruiert Prozess aus Input und Output → erstellt Schritt-für-Schritt-Todo → wir gehen Teile live durch → Skill-Creator macht daraus eine wiederverwendbare Skill.

===

<!-- @type: content -->
## Schluss — Von der Frage zum Audio-Briefing

### Use Case
"Der Report von vorhin — jetzt für die Autofahrt."

### Features
**Deep Research** (aus Demo 1) · **NotebookLM**

### Live
Deep-Research-Ergebnis öffnen → an NotebookLM übergeben → Podcast generiert, während wir alle vier Demos kurz rekapitulieren → 2–3 Minuten Podcast anhören.
