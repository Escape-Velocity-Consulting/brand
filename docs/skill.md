# Brand skill

The `escape-velocity-brand` skill is a Claude Code / Desktop skill that teaches Claude the brand design system AND ships the canonical assets — templates, tokens, fonts, logos, the brand reference, the canonical spec, and a web-prototyping kit. It is the primary on-brand asset creator. The MCP server is an optional extension.

## Architecture: author + optional renderer

The skill is the brand's primary asset creator. It ships everything Claude needs to produce on-brand HTML, SVG, and prototypes standalone — no network, no MCP, no Playwright. The `escape-velocity-brand` MCP server (see [mcp-server.md](mcp-server.md)) is an **optional extension** that adds two capabilities the skill can't provide on its own:

- **Pixel rendering** — Playwright-driven HTML → PNG / HTML → PDF
- **Publishing** — promotes a render to a permanent public URL on the Brand Site

When the MCP is registered, prefer it for binary outputs (PDF, PNG). When it isn't, the skill authors finished HTML from the same canonical templates and hands it to the user — they save the file and open it in a browser.

This is a deliberate split. The skill ships the **knowledge and source materials** (templates, tokens, fonts, brand spec). The MCP ships the **rendering and publishing infrastructure**. Anything the skill can do without Playwright is done by the skill.

## Edit flow

Source files live in the brand repo. **Never edit the deployed copy** at `.claude/skills/escape-velocity-brand/` — it gets overwritten on reinstall.

| What changed | Edit | Then |
|--------------|------|------|
| Skill instructions (hand-authored sections) | `skill/escape-velocity-brand/SKILL.md` | `npm run build:skill` → reinstall |
| Brand reference sidecar | `BRAND_SKILL.md` (repo root) | `npm run build:skill` (copies to `references/brand-reference.md`) |
| Canonical brand spec | `BRAND_SPEC.md` (repo root) | `npm run build:skill` (copies to `references/brand-spec.md`) |
| Template registry (catalog + routing blocks in SKILL.md) | `templates.meta.ts` | `npm run build:skill` regenerates auto-blocks + emits JSON copy |
| Template HTML | `templates/<key>.html` | `npm run build:skill` (re-stages into bundle) |
| Tokens (colors, fonts, spacing) | `tokens.ts` | `npm run build:skill` (rebuilds `tokens.css` + `tokens.json` and re-stages) |
| Logo SVGs | `assets/logos/*.svg` | `npm run build:skill` (re-stages) |
| Web prototyping CSS | `site/site.css`, `site/print.css`, `web/starter.html` | `npm run build:skill` (re-stages into `web/`) |
| Press boilerplate | `press/boilerplate.md` | `npm run build:skill` (re-stages) |

`npm run build:skill` produces `dist/escape-velocity-brand.skill`. Install in Claude Code/Desktop via the skill install flow.

## Auto-generated sections in SKILL.md

Two blocks are mechanically regenerated by `scripts/build-skill-catalog.ts`:

- `<!-- AUTO-GENERATED:CATALOG -->` … `<!-- /AUTO-GENERATED:CATALOG -->` — the template catalog
- `<!-- AUTO-GENERATED:ROUTING -->` … `<!-- /AUTO-GENERATED:ROUTING -->` — the routing table

**Hand-edits inside these markers are lost on the next `build:skill` run.** Edit `templates.meta.ts` instead. See [templates.md § The propagation chain](templates.md#the-propagation-chain).

The same script, invoked with `--emit-json <path>`, also writes `references/templates.meta.json` into the skill bundle so Claude can read the registry as plain data (no `tsx` runtime available inside the skill).

## Skill-only mode (MCP not registered)

The skill is fully usable without the MCP. When asked to produce a branded asset:

1. **Locate the template** — read `templates/<key>.html` directly from the bundle.
2. **Resolve inheritance** — for document templates (letter/offer/invoice/tos/report) that use `{% extends "_base.html" %}`, also read `_base.html` and `_recipient.html`. Slot the child's `{% block ... %}` content into the base's matching block placeholders.
3. **Substitute variables** — replace `{{ TOKENS_CSS | safe }}` with the contents of `tokens/tokens.css`, `{{ FONTS_URI }}` with `./fonts`, and any `{{ VAR }}` placeholders with provided values.
4. **Hand the user the HTML** — return as text with a suggested filename. Tell them to save it next to a copy of the bundle's `fonts/` folder if they want brand fonts to load (otherwise the page falls back to system fonts).

This works for all template categories: documents, social images, carousel slides, presentations, palette sheets. The output is a single HTML file the user can open in any browser.

## What the skill cannot do without the MCP

- **PNG / PDF binary rendering** — requires Playwright. The skill hands the user HTML; the user needs the MCP (or any Chromium-based tool) to get binaries.
- **Publishing to the Brand Site** — requires server-side storage. There is no local equivalent.

## Routing behavior

The skill's routing table (auto-generated from `templates.meta.ts` `prompts` fields) maps natural-language inputs to template tools. When the MCP is registered, the right column is a direct tool call. When it isn't, swap to skill-only authoring: read the same template from `templates/<key>.html` and substitute variables manually.

Example excerpts:

| Phrasing | With MCP | Without MCP |
|---|---|---|
| "Brief, letter, Korrespondenz, briefing" | `render_template({ template: 'letter', markdown, ... })` | read `templates/letter.html` + `_base.html` + `_recipient.html`, substitute, hand user HTML |
| "OG image, blog share image" | `render_template({ template: 'social/og', vars })` | read `templates/social/og.html`, substitute vars |
| "LinkedIn carousel, multi-slide post" | `render_slides({ pages: [...], ... })` | fill each `templates/carousel/*.html`, hand user N HTML files |
| "publish this", "make it permanent" | `publish_artifact({ bundleId })` | not available — only MCP path |

**Adding a new template phrase:** don't edit SKILL.md by hand. Edit the `prompts: [...]` array in the template's `templates.meta.ts` entry and run `npm run build:skill`. See [templates.md § Adding a new template](templates.md#adding-a-new-template--step-by-step).

## Mental model the skill teaches Claude

1. **The skill is the author. The MCP is the renderer.** Read templates from your own bundle. Hand the user finished HTML. Use the MCP for binaries when registered.
2. **Templates first, custom HTML last.** If a template matches the user's intent, use it. Custom HTML is the last resort, only when nothing matches.
3. **Tokens are CSS vars.** When authoring HTML, use `var(--color-terracotta)` / `var(--font-headline)` etc. — never hex values or font names. See [rules.md § Token-Sourcing Rule](rules.md#token-sourcing-rule).
4. **Publish is deliberate.** Renders are ephemeral by default (1h TTL). Only publish when the user explicitly asks. See [publishing.md](publishing.md).
5. **Don't recreate templates from memory.** The canonical files ship with the skill — read them.

## Anti-patterns

- ❌ Editing the deployed `.claude/skills/escape-velocity-brand/` directly — your changes vanish on the next reinstall.
- ❌ Hand-editing inside `<!-- AUTO-GENERATED:* -->` markers — they get rewritten.
- ❌ Adding a new template phrase only to SKILL.md (skipping `templates.meta.ts`) — the catalog regenerates from the registry.
- ❌ Treating SKILL.md and BRAND_SKILL.md as the same file. They're separate sources; both must be updated when content overlaps.
- ❌ Authoring brand HTML from memory when a template exists. Read `templates/<key>.html` — the canonical file ships with you.
