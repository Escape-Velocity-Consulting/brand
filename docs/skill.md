# Brand skill

The `escape-velocity-brand` skill is a Claude Code / Desktop skill that teaches Claude the brand design system + tool-routing logic. It's the bridge between "user says: write a letter to Validvent" and "Claude calls the right `render_template` with the right args."

## Phase 3 redesign — thin guidance only

The skill bundle is **~8KB**. It does *not* ship generators, templates, fonts, tokens, or a `setup.sh` bootstrap. All rendering is delegated to the escape-velocity-brand MCP server (see [mcp-server.md](mcp-server.md)). The skill teaches Claude the design system + routing; the MCP does the pixel work.

This is a deliberate split:

- Before Phase 3 the skill carried a copy of the templates and ran generators locally. That meant every brand update required a skill rebuild + reinstall on every workstation.
- After Phase 3 the skill just describes the system. The actual templates live in the brand repo; the MCP server reads them at render time.

## Bundle contents

```
escape-velocity-brand.skill (zip)
├── SKILL.md                              ← mental model, 9-tool reference, routing decision tree, HTML authoring rules, examples
└── references/
    └── brand-reference.md                ← token table + design patterns (sourced from BRAND_SKILL.md)
```

## Edit flow

Source files live in the brand repo. **Never edit the deployed copy** at `.claude/skills/escape-velocity-brand/` — it gets overwritten on reinstall.

| What changed | Edit | Then |
|--------------|------|------|
| Skill instructions (hand-authored sections) | `skill/escape-velocity-brand/SKILL.md` | `npm run build:skill` → reinstall |
| Brand reference sidecar | `BRAND_SKILL.md` (repo root) | `npm run build:skill` (copies to `references/brand-reference.md` at staging) |
| Template registry (catalog + routing blocks in SKILL.md) | `templates.meta.ts` | `npm run build:skill` regenerates the auto-generated blocks |

`npm run build:skill` produces `dist/escape-velocity-brand.skill`. Install in Claude Code/Desktop via the skill install flow.

## Auto-generated sections in SKILL.md

Two blocks are mechanically regenerated by `scripts/build-skill-catalog.ts`:

- `<!-- AUTO-GENERATED:CATALOG -->` … `<!-- /AUTO-GENERATED:CATALOG -->` — the template catalog
- `<!-- AUTO-GENERATED:ROUTING -->` … `<!-- /AUTO-GENERATED:ROUTING -->` — the routing decision tree

**Hand-edits inside these markers are lost on the next `build:skill` run.** Edit `templates.meta.ts` instead. See [templates.md § The propagation chain](templates.md#the-propagation-chain).

## Graceful degradation

The skill explicitly handles the "escape-velocity-brand MCP not registered" case. If the MCP is unavailable, Claude can still:

- Author on-brand HTML using token CSS variables (from the embedded `brand-reference.md`)
- Hand the HTML back to the user as text
- Let the user register the MCP later for pixel-perfect rendering

This makes the skill usable in environments where the MCP isn't wired up yet.

## Routing behavior

The skill's decision tree (auto-generated from `templates.meta.ts` `prompts` fields) maps natural-language inputs to the right tool. Example excerpts:

| Phrasing | Tool call |
|---|---|
| "Brief, letter, Korrespondenz, briefing, Anschreiben" | `render_template({ template: 'letter', markdown, ... })` |
| "Slide deck, presentation, Foliendeck" | `render_slides({ markdown, dimensions: 'slide-16-9', outputs: { viewer: true, pdf: true } })` |
| "LinkedIn carousel, multi-slide post" | `render_slides({ pages: [...], dimensions: 'linkedin-portrait', outputs: { pdf: true, pngs: true } })` |
| "publish this", "make it permanent", "put it on the site" | `publish_artifact({ bundleId })` |
| "unpublish &lt;id&gt;" | `unpublish_artifact({ id })` |

**Adding a new template phrase:** don't edit SKILL.md. Edit the `prompts: [...]` array in the template's `templates.meta.ts` entry and run `npm run build:skill`. See [templates.md § Adding a new template](templates.md#adding-a-new-template--step-by-step).

## Mental model the skill teaches Claude

1. **Templates first, custom HTML last.** If a template matches the user's intent, use it. Custom HTML via `render_html_to_png` / `render_html_to_pdf` is the last resort, only when nothing in the catalog matches.
2. **Tokens are CSS vars.** When authoring HTML, use `var(--color-terracotta)` / `var(--font-headline)` etc. — never hex values or font names. See [rules.md § Token-Sourcing Rule](rules.md#token-sourcing-rule).
3. **Publish is deliberate.** Renders are ephemeral by default (1h TTL). Only publish when the user explicitly asks. See [publishing.md](publishing.md).
4. **Get the user's content right before rendering.** The skill emphasizes drafting + iterating in conversation, then rendering once the content is locked.

## Anti-patterns

- ❌ Editing the deployed `.claude/skills/escape-velocity-brand/` directly — your changes vanish on the next reinstall.
- ❌ Hand-editing inside `<!-- AUTO-GENERATED:* -->` markers — they get rewritten.
- ❌ Adding a new template phrase only to SKILL.md (skipping `templates.meta.ts`) — the catalog regenerates from the registry, so your change is wiped on the next build.
- ❌ Treating SKILL.md and BRAND_SKILL.md as the same file. They're separate sources; both must be updated when content overlaps.
